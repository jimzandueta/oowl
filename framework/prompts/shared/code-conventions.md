# Code Conventions

Customize this file with your project's coding conventions.

Examples of what to define:
- Naming conventions (casing, prefixes, file naming)
- Import ordering and module structure rules
- Code formatting preferences (max line length, indentation)
- Comment style and documentation expectations
- Test naming and structure conventions
- Deprecation and cleanup practices

Keep this file version-controlled. It is loaded into every implementation agent's context.
